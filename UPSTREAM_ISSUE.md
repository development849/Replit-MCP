# Suggested issue text for upstream NOVA-3951/Replit-MCP

Title: Add a LICENSE file to clarify MIT licensing

Body:

> Hi! Thanks for publishing this server — it's a useful starting point. The `package.json` declares `"license": "MIT"`, but the repo doesn't ship a LICENSE file. GitHub's repo info therefore reports the license as "None", which technically defaults to all-rights-reserved and makes downstream forks legally ambiguous.
>
> Could you add a top-level `LICENSE` file with the standard MIT text and your name on the copyright line? That would match the package.json declaration and let people fork/redistribute confidently. Happy to open a PR if helpful.
